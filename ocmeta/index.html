  • Converter

    Class to clean csv:

    • clean_id(self)

      remove spaces between main BR's IDs and between ":", it returns a space-separated IDs list + MetaID
      It handles the situation when a BR already exists in the csv, giving it the same meta ID.

    • clean_ra(self)

      remove spaces around ra name (or around name and surname separated by "," if ra is an author/editor) and it removes spaces between ra's IDs and between their ":", it returns "name, surname" (or Publisher's name) plus, between square brakets, a space-separated IDs list + MetaID
      It handles the situation when an RA already exists in the csv, giving it the same meta ID.

    • clean_vvi(self)

      remove spaces around venue,volume or issue's br-name and it removes spaces between venue's IDs and between their ":", it returns "br-name" plus, between square brakets, a space-separated IDs list (in case of Venue) + MetaID.
      It handles the situation when a BR already exists in the csv, giving it the same meta ID.
      It also takes in consideration the relationship between Venue-Volume-Id in order to give the correct ID (i.e. same Volume of a Venue)

Week_8 Report
DONE
  • Converter
    • finder.py

      Ancillary script to execute complex SPARQL query on a triplestore
      Since I had to write from scratch all functions i prefered to create a new script istead of working on resfinder.py
      finder.py works with pymantic library to handle SPARQL queries, and it is based mainly on 3 functions:

      • retrieve_br_from_id

        Given an ID retrieve the BR, his Title and all his IDs

      • retrieve_publisher_from_id and retrieve_autor_editor_from_id

        Given an ID retrieve the RA, his Name (name and surname if person) and all his IDs

      • retrieve_venue_from_id and retrieve_vvi_part_from_id

        Given an ID retrieve the Venue, his title and all his IDs, and then in a recursive way (if available) all the volumes, issues and also issues inside volume

    • Real Time Csv Entities Update

      Storing all the info about entities and thanks to a csv's lines counter Converter can update any entity when a new ID is found. It is important to notice that a BR can be inside "ID" or inside "VENUE"

    • clean_title and clean_name

      Usually a BR title or RA name is taken from Triplestore or Local memory (dictionary) when available. But, when we find a new name/title a script cleans it

      • Name: strip by "," (only for people: authors, editors) and then strip by whitespaces (one or more, in order to handle multiple names) then capitalize every word (first letter upper, the rest lower). Finally join with a single whitespace.
      • Title: strip by whitespaces (one or more) then for every word, if there's an uppercase character pass, else capitalize (first letter upper, the rest lower). Finally join with a single whitespace.
    • Counter

      Now Converter's counter works exactly as Graphlib (same code, how to give author's attribution inside code?)

    • Ready to work

      Script to generate Id's Indices CSV, it's commented waiting for the final implementation
